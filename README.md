# originlab-voltammetry-peaks
Python automation for OriginLab to automatically detect and calculate peak heights from voltammetry data Gammry devices, as an example here of Linear sweep voltammetry

## üåü Key Features

The script executes the following analysis steps for each voltammetry cycle:

* **Baseline Estimation:** Accurately estimates the electrochemical background current.
* **Peak Definition:** Automatically identifies the potential and current of the maximum peak.
* **Peak Height Calculation:** Calculates the true peak height relative to the estimated baseline.
* **Statistical Summary:** Computes the **mean** and **standard deviation (SD)** of peak heights from multi-cycle experiments.
* **Data Visualization:** Generates a clearly **illustrated graph** within OriginLab showing the detected peak and baseline.

---

## üìä Input Data Format

The script is configured to process one or more **Linear Sweep Voltammetry (LSV)** raw data files (typically **\*.DTA** format) generated by a Gamry Potentiostat.

`C:\Users\ <your_username> \Documents\OriginLab\User Files`

## ‚ñ∂Ô∏è How to Run

Follow these steps to run the automation script in OriginLab:

1.  **Import Data:** Use the **"Import Gamry Voltammetry Files"** App to load your **\*.DTA** file(s) into OriginLab.
2.  **Open Command Window:** Press **`Alt + 3`** or navigate to **Window** $\rightarrow$ **Command Window**.
3.  **Execute Script:** In the Command Window, type the following command and press **Enter**:
    ```
    run -pyf findpeak.py
    ```
4.  **Define Baseline Range:** The script will prompt you in the Command Window to input the **Initial** and **Final Potential (in Volts, V)** that define the region used for **baseline estimation**.

---

## üìà Output

Upon completion, the script generates the following analytical outputs directly within your OriginLab project:

* **Results Note:** A comprehensive note sheet containing:
    * Individual peak heights.
    * The **average** peak height across all cycles.
    * The **standard deviation (SD)** of the peak heights.
* **Illustrated Graph:** A new graph window visualizing the imported data, clearly marking the detected **peak position** ($E_p, I_p$) and the calculated **baseline** used for the measurement.
